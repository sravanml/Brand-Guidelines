When to use each color

Understanding the role of each color ensures consistency across every touchpoint — web, print, social media, and events.

Royal Blue
#1A5EA8 · Primary
  • Nav Navigation bar background, header, and sticky top bar on every page
  • CTA Primary call-to-action buttons — Register, Submit, Learn More
  • Links All hyperlinks, interactive text, and inline actions across web
  • Headings Section titles, card headings, and display text (Blue 800)
  • Print Report covers, event banners, letterheads, and brochure headers
  • Tint Blue 50 for info panel backgrounds, table rows, subtle section fills
✕ Do not use as a background for large body content areas — reserve for structural and interactive elements only
Saffron Orange
#F7941D · Accent
  • Accent Secondary buttons, badges, tags, and decorative highlights alongside blue
  • Icons Icon fills, bullet markers, and small decorative UI elements
  • Dividers Ornamental section dividers, quote borders, and card accent bars
  • Eyebrow Section labels, category tags, and overline text above headings
  • Print Poster highlights, pull-quote borders, and event programme accents
  • Tint Saffron 50 for quote card fills, notification backgrounds, warm highlights
✕ Never use as the dominant color or equal weight to blue — saffron supports, it does not lead
Warm Neutral
#F9F7F4 → #342C22 · Foundation
  • Page bg Neutral 50 (#F9F7F4) for every page background — warm, never cold white
  • Cards Pure white (#FFFFFF) for card surfaces, modals, and content panels
  • Body text Neutral 800 for primary copy, Neutral 600 for secondary and captions
  • Borders Neutral 200 for dividers, card borders, input outlines across all surfaces
  • Disabled Neutral 300–400 for disabled states, placeholder text, and inactive items
  • Print Neutral 800 for all body copy in documents, reports, and publications
✕ Never use pure black (#000000) or cold grays — they break the warm tone of the palette
